# Password Security

Password security ka matlab hai passwords ko strong aur secure rakhna
taake attackers unhein easily guess ya crack na kar saken.

## What is a Strong Password?

Strong password wo hota hai jo:
- Kam az kam 8â€“12 characters ka ho
- Capital aur small letters hon
- Numbers aur symbols hon
- Guess karna mushkil ho

Example:
- Weak: 123456
- Strong: A9@kP#21!

## Risks of Weak Passwords

- Accounts easily hack ho sakte hain
- Data leak ho sakta hai
- Identity theft ka risk hota hai

## What is Hashing?

Hashing ek process hai jisme password ko
ek fixed-length code (hash) mein convert kar diya jata hai.

- Original password wapas nahi milta
- Same password ka same hash hota hai

Examples:
- MD5
- SHA-256

## Hashing vs Encryption

Hashing:
- One-way process hota hai
- Passwords store karne ke liye use hota hai

Encryption:
- Two-way process hota hai
- Data ko encrypt aur decrypt kiya ja sakta hai

## Best Practices

- Same password multiple sites par use na karo
- Two-Factor Authentication (2FA) use karo
- Password manager ka use

## Key Takeaways

- Strong passwords security ko improve karte hain
- Hashing passwords ko safely store karti hai
- Weak passwords major security risk hain
